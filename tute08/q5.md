```mermaid
sequenceDiagram
    participant Client
    participant Server
    participant Database

    activate Client
    Client->>Server: login(username, password)
    activate Server
    Server->>Database: requestData(username, password)

    alt Data Is Corrupted
        Database-->>Server: Corruption Exception
        Server-->>Client: Bad Credentials
    else Data Is Not Corrupted
        Database-->>Server: data
        Server->>Server: processData(data)
        alt Data Doesn't Match
            Server-->>Client: Invalid Credentials
        else Data Matches
            Server->>Server: updateLogin(username)
            Server-->>Client: Login Successful
        end
    end
    activate Database
```

Client tries to login
Server will ask database for credentials
If data exists, server will confirm credentials and try login
If data not match credentials, server will not login
If data is corrupted, or the database screams at the server, server will scream at the user in turn with an exception
